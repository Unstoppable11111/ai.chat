## Project Overview

这个概念把内容规划、实验追踪和项目执行整合进一个工作台，面向的是需要效率和视野兼顾的 solo operator。

## Problem

当提示词、笔记、参考图和任务分散在不同地方时，最大的损耗不是时间，而是上下文切换。

## Approach

我把它设计成一层安静的运营界面：信息密度够高，但结构必须足够清楚。

## Design System

暗色面板、基于 token 的间距和高对比排版，让界面看起来锋利但不刺眼。

## Tech Stack

Next.js、TypeScript、Tailwind、MDX，再加一层轻量自动化，用于做资料整理、提示词分类和内容初稿。

## Result

这个系统能更清楚地回答三件事：现在在做什么、什么值得复用、什么已经接近可上线。

## Next Step

把实验结果直接连到学习资料和构建日志里。
