## Project Overview

这个 workbench 把提示词当作资产管理，而不是聊天记录残留。它为每条提示词提供结构、标签和示例图。

## Problem

提示词质量会复利，但前提是检索足够容易。很多创作者仍然在靠记忆找 prompt。

## Approach

界面重点放在搜索、筛选和快速复制动作上，让“使用”本身成为页面的核心。

## Design System

卡片式结构、清晰分组、克制发光和示例图作为视觉锚点。

## Tech Stack

Next.js 负责应用壳，MDX 负责补充说明，结构化 mock data 负责早期验证。

## Result

这个概念证明了提示词管理可以更像产品基础设施，而不是笔记整理。

## Next Step

为每条提示词增加变体、生成结果和失败案例。
