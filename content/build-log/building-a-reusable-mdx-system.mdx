## 一套管道，三种内容

实验记录、构建日志和项目案例都需要元信息、可读布局和结构化章节。MDX 是现在这个阶段最轻、最够用的方案。

## 目录的意义

目录不只是为了跳转，它会让详情页看起来更像工作文档，而不是普通博文。

## 代码高亮

虽然代码主要出现在 Build Log，但统一的高亮风格能让整个网站更稳定。

```ts
const toc = source
  .split("\n")
  .filter((line) => line.startsWith("##"));
```

## 为什么它能扩展

现在新增内容基本只需要加一个文件和 frontmatter。这种低摩擦，正是这个网站需要的。
