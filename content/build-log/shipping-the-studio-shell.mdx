## 为什么先做壳子

我希望这个网站从第一天起就不是“内容列表”，而是一个真的能承载工作流的系统。首页应该像控制台，而不是文章流。

## 第一原则

它需要有创业者的工作感，但不能吵。暗色面板、克制对比、明确结构，比炫技更重要。

## 实现选择

我选择 App Router + 内容驱动页面结构，因为实验记录、构建日志和项目案例本质上是三种内容系统，只是共享同一个视觉框架。

```tsx
export const metadata = {
  title: "Chen Tech Studio",
  description: "A personal tech portfolio",
};
```

## 这一步学到什么

当我不再把首页当 landing page，而把它当作个人站点的记录面板来设计时，整个站点的方向就清晰了。
